# ЁЯЪА azurelab-autopilot

> **PowerShell + Terraform autopilot** to deploy a Windows Server VM on Azure тАФ fully interactive, no manual `.tfvars` editing required.

---

## ЁЯУЛ Overview

`azurelab-autopilot` is a PowerShell-driven wizard that:

- Connects to Azure interactively
- Collects all VM parameters via **GUI dropdowns and prompts**
- Auto-generates `terraform.auto.tfvars`
- Runs `terraform init / validate / plan / apply` in one shot

No pre-filled config files. No hardcoded secrets. Just run and answer.

## ЁЯзн Wizard Steps

Once launched, the script guides you through:

1. **Azure login** тАФ `Connect-AzAccount` with your Tenant + Subscription
2. **Resource Group name** тАФ Terraform will create it (warns if it already exists)
3. **VM name** and **Private IP** (static)
4. **Location** тАФ searchable dropdown (e.g. `westeurope`)
5. **VM family** тАФ A / B / D / DC family selector
6. **VM size** тАФ filtered dropdown with vCPU and RAM info (e.g. `Standard_B2ms`)
7. **OS disk** тАФ type (Standard_LRS / StandardSSD_LRS / Premium_LRS) + size in GB
8. **Data disk** тАФ type + size (enter `0` to skip)
9. **Public IP** тАФ yes/no prompt
10. **Admin credentials** тАФ username + password (validated: 12+ chars, upper, lower, digit, special)
11. **Terraform action**:
    - `0` тАФ generate tfvars only, no Terraform run
    - `1` тАФ `init` + `plan`
    - `2` тАФ `fmt` + `init` + `validate` + `plan` + `apply` (requires typing `APPLY` to confirm)

The script auto-generates `terraform/terraform.auto.tfvars` тАФ **never commit this file**.

## ЁЯЦея╕П What Gets Deployed

- **Resource Group**
- **Virtual Network** (10.0.0.0/16) with subnet `snet-test` (10.0.1.0/24)
- **Windows Server 2022 Datacenter Gen2** VM
- **Static private IP** NIC
- **OS disk** + optional **data disk**
- Optional **Public IP**
- **DNS servers**: VM private IP + Azure default (168.63.129.16)

## ЁЯФТ Security Notes

- `terraform.auto.tfvars` contains your **admin password in plaintext** тЖТ excluded from git via `.gitignore`
- `*.tfstate` files may contain sensitive data тЖТ **never commit them**
- For production use, store state in a remote backend (Azure Storage Account)
- `*.json` files are broadly ignored тАФ verify no legitimate config files are excluded in your case
